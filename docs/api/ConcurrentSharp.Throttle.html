Class Throttle

Instances of the Throttle class can be used to restrict the number of simultaneous operations taking place using the same throttle.

Inheritance
System.Object
Throttle
Namespace: ConcurrentSharp
Assembly: cs.temp.dll.dll
Syntax
public sealed class Throttle : IDisposable
Remarks

To use a Throttle create an instance specifying the maximum number of concurrent operations you want to occur. Then have each operation call Enter() when it begins and capture the returned token. When the operation completes, call Dispose(). The Throttle will ensure no more than the specified number of operations run at the same time.

Examples
var throttle = new Throttle(4); //No more than 4 simultaneous operations.

// Start 100 calls to a 'DoWork' function.
for (int cnt = 0; cnt < 100; cnt++)
{
    System.Threading.ThreadPool.QueueUserWorkItem
    (
        (reserved) => 
        {
            using (var token = throttle.Enter()) //Obtain a token from the throttle instance to restrict the concurrent jobs
            {
                DoWork(throttle);
            } // Token will be disposed here, allowing another job to run if one is waiting.
        }
    );
}

Constructors

Name Description
Throttle(Int32)

Constructs a new Throttle instance that allows up to the specified number of concurrent operations.

Methods

Name Description
Dispose()

Disposes this throttle and all internal resources.

Enter()

Creates and returns a ThrottleToken that can be used to notify this throttle that a job has completed. If there are already the maximum number of operations running using this throttle, then this method will block the calling thread until another job completes.
Execute(Action)

A convenience method that executes the action operation using the throttle, eliminating manual management of the throttle token.
Execute<T>(Action<T>, T)

A convenience method that executes the action operation using the throttle, eliminating manual management of the throttle token.
Execute<TResult>(Func<TResult>)

A convenience method that executes the func operation using the throttle and returning it's result while eliminating manual management of the throttle token.
Execute<T, TResult>(Func<T, TResult>, T)

A convenience method that executes the func operation using the throttle and returning it's result while eliminating manual management of the throttle token.
ExecuteAllAsync<T>(IEnumerable<T>, Func<T, Task>)

Executes asyncAction over all items in items, using the throttle to restrict the maximum number of parallel executing tasks.
ExecuteAllAsync<TArg, TResult>(IEnumerable<TArg>, Func<TArg, Task<TResult>>)

Executes asyncAction over all items in items, using the throttle to restrict the maximum number of parallel executing tasks.
